# KMT Legal Copy Style Guide

## القاعدة العامة

- العربية هي لغة المنتج الأساسية في الموقع العام ولوحة الإدارة وبوابة العميل.
- استخدم صياغة رسمية واضحة ومباشرة تناسب مدير مكتب أو عميل غير تقني.
- لا تعرض أكواد داخلية أو JSON أو أسماء صلاحيات تقنية للمستخدم إلا داخل سياق تشخيصي واضح.

## ما يبقى بالإنجليزية

- أسماء البراند والمزودين: `KMT Legal`, `SMTP`, `OpenRouter`, `PostgreSQL`.
- القيم التقنية اللازمة للتشخيص أو الربط: البريد الإلكتروني، الروابط، أرقام الفواتير، الأكواد، و`Slug`.
- أسماء ملفات أو أنواع ملفات مثل `PDF`, `DOCX`, `PNG`.

## تسميات موحدة

- `Super Admin`: مدير النظام
- `Office Admin`: مدير المكتب
- `Marketing Staff`: مسؤول التسويق
- `Lawyer`: محام
- `Client`: عميل
- `manual`: يدوي
- `read-only`: للقراءة فقط
- `Slug`: معرّف الرابط (Slug)

## رسائل الأخطاء

- رسائل `error.message` التي تظهر للمستخدم تترجم من خلال `src/lib/ui-copy.ts`.
- أبق `error.code`, `details`, و`requestId` كما هي لأنها مفيدة للتشخيص والدعم.
- لا تضع بيانات شخصية أو أسرار داخل رسائل الخطأ.

## النص المختلط والاتجاه

- استخدم `dir`, `bdi`, أو `ltr inline-block` للأرقام، البريد، الروابط، UUID المختصر، وأرقام الفواتير داخل النص العربي.
- لا تترجم القيم المخزنة في قاعدة البيانات عند الإرسال للـAPI؛ ترجم العرض فقط.
